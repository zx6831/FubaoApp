# 福豹慢病共管家庭智能体：iOS 与云端 V1 设计

**日期：** 2026-07-16  
**状态：** 已确认，待规格文档审阅  
**交付目标：** 面向中国大陆用户的 Flutter iOS 单应用（子女端、长辈端双角色）与可部署云端后端。首期使用模拟福豹设备，发布前支持真实 iPhone 测试和 TestFlight。

## 1. 产品边界

福豹是“健康管理与关怀提醒”产品，不作诊断、治疗、药物剂量建议或医疗器械测量声明。

- 健康数据来自用户手动确认/录入，或未来接入的真实设备。
- 血压、血糖等异常只触发分级关怀提醒；高风险状态提示用户联系医生或当地急救服务。
- V1 不使用手机传感器测量健康指标。
- App 使用一个 iOS 安装包；登录后按家庭角色进入子女端或长辈端，不允许跨角色越权操作。

## 2. 技术架构

### 2.1 客户端

- Flutter 客户端，目标平台 iOS；架构预留 Android 扩展能力。
- 子女端和长辈端使用独立导航和组件规范。
- 长辈端采用大字体、最小 60px 触控区、简短文案、选择式任务卡和一键朗读。
- 客户端通过 HTTPS 调用后端 API；缓存最近一次首页和任务数据以支持网络异常降级。

### 2.2 后端

- NestJS REST API，自动输出 OpenAPI 文档。
- PostgreSQL 存储业务数据，Redis 保存验证码、邀请有效期、会话辅助数据与异步队列。
- 后台任务负责每日任务生成、话题生成、未完成提醒和告警处理。
- iOS 通知使用 APNs；短信供应商、真实设备 MQTT 供应商以适配器形式接入。
- 初期提供模拟福豹设备服务，模拟发现、配网、在线、任务提醒、确认和健康数据上报。真实硬件通过兼容 MQTT 协议替换模拟实现。

## 3. 权限与主要业务

### 3.1 家庭与权限

- 一家庭仅支持 1 名子女主账号和 1 名长辈账号。
- 子女创建家庭并生成 4 位邀请码，有效期 30 分钟；长辈输入邀请码加入。
- 所有家庭、任务、健康数据和告警查询均由服务端按家庭关系做二次鉴权。

### 3.2 V1 功能

1. 手机号验证码登录；测试环境提供测试验证码。
2. 模拟设备激活：发现、配网、在线、激活状态。
3. 健康档案：慢病类型、亲属称呼、紧急联系人、授权信息；支持查看、导出与删除申请。
4. 子女配置用药、测量、运动与生活习惯计划；服务端每日生成任务。
5. 长辈通过“已完成/没做”选择式卡片执行任务；子女可查看今日、周视图、月历与详情。
6. 健康火花：长辈当天完成至少一项任务且子女当天打开 App 时点亮；连续 7 天双方未活跃则熄灭并重置。
7. 手动记录血压、血糖、心情等健康指标；首期仅 L1/L2 关怀提醒并记录处理结果。
8. 基于任务、火花和已确认健康数据生成模板化关怀话题；支持复制话术。
9. 设置：通知和勿扰、字体/朗读、家庭成员、设备状态、注销、数据导出与删除。

### 3.3 非目标

- 真实硬件 BLE/MQTT 接入、L3 自动升级、视频、OCR、支付、多子女、Android。
- 以上能力保留为后续迭代和集成边界，不在 V1 实现范围内。

## 4. 数据与安全

核心实体：用户、会话、家庭、邀请码、健康档案、计划、每日任务、任务记录、健康指标、告警、火花日活、话题、设备状态、审计日志。

- API 全程 HTTPS；敏感字段加密存储；客户端令牌使用系统安全存储。
- 关键操作写入审计日志；注销后数据进入 30 天删除队列；提供个人数据导出。
- 测试和演示环境仅使用虚构健康数据。
- 客户端离线时显示缓存数据；网络恢复后重试。任务写入使用幂等键，避免重复完成记录。
- 推送、短信、模拟设备失败不会篡改任务真实状态。

## 5. 验收与上架准备

### 5.1 自动化测试

- 后端：邀请码、角色权限、每日任务、火花规则、告警阈值、删除流程。
- Flutter：角色导航、适老任务卡、关键状态、异常提示。
- 接口：登录、绑定、计划、任务、健康数据、告警和数据删除闭环。

### 5.2 发布门槛

- 在真实 iPhone 完成测试，随后通过 TestFlight 分发。
- 发布前提供稳定可访问的后端和 App Review 演示账号/演示模式。
- 准备隐私政策、支持网址、账号注销入口、App 图标、商店截图和准确元数据。
- 上传 TestFlight 和 App Store 需要 macOS、Xcode 与 Apple Developer 账号；当前 Windows 环境可以完成 Flutter/后端开发和自动化测试。
